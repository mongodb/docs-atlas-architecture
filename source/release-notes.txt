{+atlas-arch-center+} Release Notes

This page lists the changes introduced in each new version of the 
{+atlas-arch-center+}.

v20250411
---------

**Released 11 July, 2025**

- :ref:`arch-center-landing-zone` page: Minor update to add use cases from the :ref:`arch-center-hierarchy` page, mention that the primary is 
  mapped *randomly* to a zone, and link to a new Operational Readiness checklist. 
- :ref:`arch-center-hierarchy` page: Adds Terraform organization creation example to examples. 
- :ref:`arch-center-network-security` page: [STRAWMAN CHANGES] Adds recommendations to use the DNS seedlist connection string and limitation for changing the number of service attachments to the 
  Private Endpoints section. 
- :ref:`arch-center-compliance` page: Adds section to introduce |atlas-gov| as an |service| compliance option.
- :ref:`arch-center-high-availability` page: [STRAWMAN CHANGES]
- :ref:`arch-center-dr` page: Minor update to recommend multi region clusters and electable nodes for high availability,
  and link to other pages with existing guidance.
- :ref:`arch-center-cost-saving-config` page: Minor update to link to the :ref:`arch-center-high-availability` page for a cost-analysis table for deployment topology. 
- Operational Readiness checklist (what page??) 

- Adds the following new pages:

  - :ref:`arch-center-migration`: Describes how to migrate data from your on-premises MongoDB deployments to |service|. 
  - :ref:`arch-center-paradigms`: Top-level page to introduce and compare different |service| deployment paradigms. 
  - :ref:`arch-center-paradigms-single`: Describes use-cases and recommendations for single-region deployments. 
  - :ref:`arch-center-paradigms-multi-region`: Describes use-cases and recommendations for multi-region deployments. 
  - :ref:`arch-center-paradigms-global`: Describes use-cases for global deployments.
  - :ref:`arch-center-paradigms-multi-cloud`: Describes use-cases and recommendations for multi-cloud deployments. 
  - :ref:`arch-center-paradigms-hybrid`: Describes use-cases for hybrid deployments.  

v20250317
---------

**Released 17 March, 2025**

- :ref:`arch-center-hierarchy` page: Minor update to add
  guidance for when to consider multiple {+clusters+} per project
  and managing local {+service+} deployments using containers.
- :ref:`arch-center-high-availability` page: Minor update to add
  more guidance for high availability and link to other pages with
  existing guidance.
- :ref:`arch-center-automation` page: Minor update to explain the 
  advantages of using {+service+}'s {+iac+} tools.
- :ref:`arch-center-automation` page: Minor update to add info about
  dashboards.

v20250228
---------

**Released 28 Feb, 2025**

- Releases the first version of the {+atlas-arch-center+}, including
  guidance to:

  - :ref:`Design a landing zone <arch-center-landing-zone>`.
  - Set up a single-region, single-cloud {+service+} architecture in
    alignment with the {+waf+}, including example commands for the
    {+atlas-cli+} and Terraform. To get started, see :ref:`arch-center-hierarchy`.
